# Контрагент																		
Раздел наполняется  

Контрагент – основная сущность для реализации логики работы B2B клиентов в Личном Кабинете.  
Обеспечивает хранение и отображение информации о контрагенте, а также связи с другими сущностями, необходимые для обеспечения основных сценариев работы Личного Кабинета.  
Вся информация, отображаемая в публичной части Личного Кабинета для авторизованного пользователя, относится к конкретному контрагенту.  
Контрагент обладает набором параметров, необходимых для отображения информации в публичной части системы (Наименование) и реализации обменных процессов (идентификатор 1С АСКУ), а также для определения доступных продуктов, складов отгрузки и ответственных менеджеров со стороны «Газпромнефть - смазочные материалы».  
Данные о контрагентах поступают в ЛКК из системы АСКУ в обменном процессе. Подробное описание обменного процесса находится в Спецификации на реализацию обменных процессов.
Управление справочником описано в ...  
Для каждого контрагента в системе заводятся пользователи.  
Менеджер КС имеет право на просмотр ЛК пользователя контрагента (порядок действий для просмотра ЛК пользователя контрагента описан в документе «ГПН_СМ_ЛК_Дистрибьютора_Руководство_пользователя_административной_части_ЛКК», в разделе 26.2).  
В режиме просмотра ЛК пользователя контрагента менеджеру КС доступны следующие разделы и их подразделы:
- Главная страница;
- Мои договоры;
- Заявки (основные и ресурсные);
- Дополнения;
- Разнарядки;
- Отчеты;
- Претензии;
- Гарантийные письма; 
- Заявки на скидки/ наценки.  
### Функциональность контрагента
Функциональность контрагента реализована в 1С Битрикс с помощью highload-блоков и стандартных модулей. Они обеспечивают:
- создание и поддержку специфической структуры данных;
- предоставление программного API для взаимодействия с данными;
- предоставление интерфейсов в системе администрирования для изменения данных;
- предоставление набора компонентов для взаимодействия с функциональностью HL-блоков и модулей на внешней части сайта.  
  
Для хранения данных контрагентов должны быть созданы или настроены следующие хранилища:
- контрагенты;
- договора;
- дополнения;
- продукция по дополнению;
